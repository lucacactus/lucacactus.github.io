Javaweb（一） - JavaWeb 简介以及 Tomcat 的使用

Author: John Doe Date: May 29, 2021  2:52:21

1 JavaWeb 的概念

  1. 什么是 JavaWeb

    JavaWeb是指，所有通过 Java 语言编写可以通过浏览器访问的程序的总称，叫 Javaweb。

    JawaWeb是基于请求和响应来开发的。

  2. 什么是请求？

    请求是指客户端给服务器发送的数据，叫请求。 Request

  3. 什么是响应？

    响应是指服务器给客户端回传数据，叫响应。 Response

  4. 请求和响应的关系

    请求和响应式成对出现的，有请求就有响应。

2 Web 资源的分类

web资源按实现的技术和呈现的效果的不同，又分为静态资源和动态资源两种。

  • 静态资源：html、css、js、txt、 mp4视频,，jpg 图片。
  • 动态资源：jsp页面、Serviet程序。

3 常用的 Web 服务器

Tomcat：由 Apache 组织的一种 Web 服务器，提供对 JSP 和 Servlet 的支持。他是一种轻量级的 JavaWeb 容器（服务器），也是当前应用最广泛的 JavaWeb 服务器（免费）。

Jboss：是一个遵从 JavaEE 规范的、开放源代码的、纯 java 的 EJB 服务器，它支持所有的 JavaEE规范（免费）。

GlassFish：由 Oracle 公司开发的一款 JavaWeb 服务器，是一款强健的商业服务器，达到产品级质量（应用很少）。

Resin：是CAUCHO公司的产品，是一个非常流行的服务器，对 Servlet 和 JSP 提供了良好的支持，性能也比较优良，resin 自身采用 Java 语言开发（收费，应用比较多）

webLogic：是 Oracle 公司的产品，是目前应用最广泛的 Web 服务器，支持 JavaEE 规范，而且不断的完善以适应新的开发要求，适含大型项目（收费，用的不多，适合大公司）。

4 Tomcat 的使用

4.1 安装

找到需要用的 Tomcat 版本对应的 zip 压缩包，解压到需要安装的目录即可。

4.2 目录介绍

  • bin 专门用来存放 Tomcat 服务器的可执行程序。
  • conf 专门用来存放 Tomcat 服务器的配置文件。
  • lib 专门用来存放 Tomcat 服务器的 jar 包。
  • logs 专门用来存放 Tomcat 服务器运行时输出的日志信息。
  • temp 专门用来存放 Tomcat 服务器运行时产生的临时数据。
  • webapps 专门用来存放部署的 web 工程。
  • work 是 Tomcat 工作时的目录，用来存放 Tomcat 运行时 jsp 翻译为 Servlet 的源码，和 Session 钝化的目录。

4.3 启动服务器

找到 Tomcat 目录下的 bin 目录下的 startup.bat 文件，双击，就可以启动 Tomcat 服务器。

如何测试 Tomcat 服务器启动成功？

打开浏览器，在浏览器地址栏中输入一下地址测试：

  1. http://localhost:8080
  2. http://127.0.0.1:8080
  3. http://真实ip:8080

当出现 Tomcat 的启动界面时，说明启动成功。

常见的启动失败的情况有：双击 startup.bat 文件，就会出现一个小黑窗口一闪而过。

这个时候，失败的原因基本上都是因为没有配置好JAVA_HOME 环境变量。常见的JAVA_HOME 配置错误有以下几种情况：

  1. JAVA_HOME 必须全部大写。
  2. JAVA_HOME 中间必须是下划线，不是减号。
  3. JAVA_HOME 配置的路径只需要配置到 jdk 的安装目录即可，不需要带上 bin 目录。

另一种启动 Tomcat 服务器的方式：使用命令行。

  1. 打开命令行
  2. cd 到 Tomcat 的 bin 目录下
  3. 敲入启动命令：catalina run

4.4 停止服务器

  1. 点击服务器窗口的关闭按钮
  2. 把 Tomcat 服务器窗口设置为当前窗口，然后按 ctrl+c
  3. 找到 Tomcat 的 bin 目录下的 shutdown.bat 双击，就可以停止 Tomcat 服务器

4.5 修改端口号

Mysql 的默认端口是 3306

Tomcat 的默认端口是 8080

找到 Tomcat 目录下的 conf 目录，找到 server.xml 配置文件。

修改完端口号，一定要重启 Tomcat 才能生效。


平时上的网站不需要输入端口号：百度——http://www.baidu.com

HTTP 协议默认的端口号是：80。打开浏览器，直接输入 localhost，访问的就是 80 端口

4.7 直接打开 html 文件和在服务器中访问 html 的区别

  • 直接打开 html 文件，此时浏览器的地址是这个 html 文件在本地的位置，使用的是 file 协议。

    file 协议表示告诉浏览器直接读取 file:// 协议后面的路径，解析展示在浏览器上即可。

  • 如果是在浏览器地址中输入访问服务器的地址，此时使用的是 http 协议。

4.8 ROOT 的工程的访问，以及默认 index.html 页面的访问

  • 当我们在浏览器地址栏中输入访问地址如下：http://ip:port，默认访问的是 ROOT 工程。
  • 当我们在浏览器地址栏中输入的访问地址如下：http://ip:port/工程名，默认访问 index.html 页面。

5 IDEA 整合 tomcat

  1. 配置：File/settings/Build, Execution, Deployment/Application Servers —> 添加 Tomcat 服务器配置，此时创建 JavaEE 项目的时候就有 Tomcat 选项了

  2. 根据配置好的 tomcat 创建 JavaEE 项目或模块，可以使用 IDEA 提供的模板，也可以创建普通项目之后再添加Web模块。

  3. 在 WEB-INF下创建目录 lib，用来存放外部依赖的 jar 包。

  4. 整个动态 web 工程目录的介绍

    • src 用来存放自己编写的 java 源代码
    • web 目录用来存放 web 工程的资源文件，比如 html 页面，css 文件，js 文件等。
    • web/WEB-INF 目录是一个受服务器保护的目录，浏览器无法访问到此目录的内容。
    • web/WEB-INF/lib 用来存放第三方的 jar 包（具体的依赖还需自己配置）。
    • web/WEB-INF/web.xml 它是整个动态 web 工程的配置部署描述文件，可以在这里配置很多 web 工程的组件，比如：
      • Servlet 程序
      • Filter 过滤器
      • Listener 监听器
      • Session 超时
      • …

  5. 为 javaEE 程序添加第三方 jar 包

    首先要从相关渠道（例如：maven repository）下载第三方 jar 包到项目的 lib 目录下，然后为项目添加这些 jar 包。

    找到 lib 文件夹下的 jar 包，直接右键 Add as Library。或者打开项目结构菜单，手动添加Library类库。

  6. 在 IDEA 中部署工程到 Tomcat 上运行

    1. 建议修改 Web 工程对应的 Tomcat 运行实例名称。

    2. 确认 Tomcat 实例中有要部署运行的 Web 工程模块。

    3. 还可以修改 Tomcat 实例启动后默认的访问地址。

    4. 在web文件夹下创建 index.jsp 文件，用于启动服务器时访问的默认页面。

    5. 关于重启选项

      • Update resources 更新 web 工程中的资源文件到 Tomcat 运行实例中
      • Update classes and resources 更新 web 工程中的 Class 字节码和资源文件到 Tomcat 运行实例中
      • Redeploy 重新部署web模块，但是不重启 Tomcat 实例
      • Restart server 重启 Tomcat 实例，并更新模块内容

    6. 其他细节

      • Edit configurations 可以设置模块的工程路径，也就是打开浏览器输入的地址:端口号后的工程文件名。
      • Edit configurations 也可以修改 Tomcat 实例占用的端口号、运行时默认打开的浏览器、热部署等
